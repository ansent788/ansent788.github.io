Git 命令行使用

发表于 2021-09-14 更新于 2023-02-08 分类于 经验随笔

1. 简介

Git 命令行总结

2. init 初始化

  • 初始化Git代码仓库

    git init

3. config 配置

  • 查看配置

    git config --list｜-l

  • 查看系统配置

    git config --system --list｜-l

  • 查看用户（global）配置

    git config --global --list｜-l

  • 配置用户（global）用户

    git config --global user.name [your name]

  • 配置用户（global）邮箱

    git config --global user.email [your email]

  • 查看仓库配置

    git config --local --list｜-l

4. add 添加

  • 添加所有新文件

    git add .

5. commit 更改

  • 带注释添加更改

    git commit -am 'message'

  • 修改更改注释

    git commit --amend

6. remote 跟踪

  • 跟踪远程分支到指定分支

    git remote add origin/[branchName] [branchName]

  • 修改远程仓库地址

    git remote set-url origin [newUrl]

  • 查看远程仓库地址

    git remote -v

7. pull 拉取

  • 远程分支拉取 - 不变基

    git pull origin [branchName]:[branchName] --no-rebase

  • 远程分支拉取 - 变基

    git pull origin [branchName]:[branchName] --rebase

  • 允许不相关历史强制合并

    git pull origin master --allow-unrelated-histories

8. push 推送

  • 推送到指定的远程分支

    git push origin [branchName]

  • 推送到指定的远程分支 - 强制

    git push origin [branchName] --force

  • 新建并推送到远程分支

    git push --set-upstream origin [branchName]

9. reset 重置

  • 重置当前最后一次更改

    git reset --soft HEAD^

10. cherry-pick 应用更改

  • 应用指定更改到当前分支

    git cherry-pick [commitHash]

  • 最近一次更改

    git cherry-pick [branchName]

  • 多个更改

    git cherry-pick [commitHash] [commitHash] ...

  • 一段更改不包含开头

    git cherry-pick [commitHash]..[commitHash]

  • 一段更改包含开头

    git cherry-pick [commitHash]^..[commitHash]

11. stash 储藏

  • 储藏当前更改

    git stash

  • 读取储藏更改

    git stash pop

12. branch 分支

  • 查看分支

    list
    git branch -l

  • 查看分支详情

    git branch -vv

  • 创建分支

    git branch [branchName]

  • 创建分支并指定跟踪分支

    git branch [branchName] -t origin/[branchName]

  • 删除本地分支

    git branch -d|-D [branchName]

  • 按指定更改创建分支

    git branch [branch] [start point]

  • 设置跟踪的远程分支

    git branch --set-upstream-to origin/[branchName] [branchName]

  • 修改跟踪的远程分支

    git branch -u origin/[branchName] [branchName]

13. checkout 检出

  • 撤消当前分支所有修改

    git checkout .

  • 切换分支

    git checkout [branchName]

  • 创建并切换

    git checkout -b [branchName]

  • 创建并切换、设置跟踪的远程分支

    git checkout -b [branchName] -t origin/[branchName]

14. rebase 变基

  • no branch, rebasing master （当前无分支，可变基到master分支）

    git rebase --continue

15. reset 重置某次更改

  • 回滚到某次更改

    git reset commit_id

  • 此次更改之后的修改会被退回到暂存区

    git reset --soft commit_id

  • 此次更改之后的修改不做任何保留

    git reset --hard commit_id

16. rebase 重新设置基准

当两个分支不在一条线上，需要执行 merge 操作时使用该命令

  • 撤销更改

    git log // 查找要删除的前一次更改的 commit_id
git rebase -i commit_id // 将 commit_id 替换成复制的值
进入 Vim 编辑模式，将要删除的 commit 前面的 `pick` 改成 `drop`
保存并退出 Vim

  • 解决冲突

    git diff // 查看冲突内容
// 手动解决冲突（冲突位置已在文件中标明）
git add <file> 或 git add -A // 添加
git rebase --continue // 继续 rebase
// 若还在 rebase 状态，则重复 2、3、4，直至 rebase 完成出现 applying 字样
git push

17. revert 还原到更改

  • 之前的更改仍会保留在 git log 中，而此次还原会做为一次新的更改

    git log // 查找需要还原的 commit_id
git revert commit_id  // 还原到这次更改

  • merge 节点的话，则需要加上 -m 指令

    git revert commit_id -m 1 // 第一个更改点
// 手动解决冲突
git add -A
git commit -m ""
git revert commit_id -m 2 // 第二个更改点
// 重复 2，3，4
git push

18. merge 合并分支

  • 合并分支到当前分支

    git branch -a -vv // 查看所有分支是否为最新更改
// 拉取最新更改
git merge [branchName] // 合并为v祸从口出当前分支
